# Documentation Structure

## Overview

The Generate Reporting Operations Guide is organized into four primary sections.

This structure is intended to provide a consistent experience across all reporting processes and fact types.

## Documentation Framework

### 1. Overview

Provides information that applies across all reporting activities.

Includes:

* Purpose of the guide
* Audience information
* Reporting lifecycle
* Documentation structure
* Shared resources

### 2. Fact Type Reporting Guides

Provides fact type specific reporting instructions.

Each Fact Type Reporting Guide follows the same structure.

#### Section 1. Process and Project Setup

Documents planning, ownership, communication, and coordination activities.

#### Section 2. Technical Setup

Documents mappings, configurations, and technical dependencies.

#### Section 3. Reporting and Validation Cycle

Documents operational reporting procedures and validation activities.

#### Section 4. Historical Reporting and Issue Tracking

Documents reporting history, issues, resolutions, and process improvements.

### 3. Data Governance

Provides governance information used across reporting activities.

Includes:

* Roles and responsibilities
* Communication plans
* Escalation procedures
* Important links
* System architecture

### 4. System Data Diagrams

Provides architecture and data flow documentation.

Includes:

* Enterprise data flow diagrams
* Generate architecture diagrams
* Reporting process diagrams
* Fact type specific diagrams

## Fact Type Reporting Guide Template

Each organization may create only the Fact Type Reporting Guides required for its reporting environment.

Not all organizations use every fact type.

Fact Type Reporting Guides should be created only for reporting areas actively managed within Generate.

## Companion Documentation

This guide is intended to complement Generate product documentation.

System configuration instructions and application functionality should continue to be maintained within the Generate documentation library.

Operational reporting procedures should be maintained within this guide.
