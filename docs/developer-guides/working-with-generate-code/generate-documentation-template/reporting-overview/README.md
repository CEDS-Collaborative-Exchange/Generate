# Reporting Overview

## Overview

The reporting lifecycle follows a structured process that moves data from source systems through Generate and into final EDFacts reporting outputs.

Each phase includes validation and review activities to ensure data quality, reporting accuracy, and successful submission outcomes.

## Validation Checkpoints

Validation occurs throughout the reporting process to ensure reporting accuracy.

Primary validation checkpoints include:

* Source data validation
* Staging validation
* Warehouse validation
* Report validation
* Submission review

Each Fact Type Reporting Guide contains detailed validation procedures specific to that reporting process.
