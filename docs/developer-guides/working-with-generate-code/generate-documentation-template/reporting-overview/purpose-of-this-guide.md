# Purpose of This Guide

## Overview

The Generate Reporting Operations Guide provides a standardized framework for managing the migration, validation, reporting, and submission processes used to create EDFacts reporting outputs through Generate.

This guide is designed to help organizations document how data moves from source systems through Generate and into reporting outputs used for federal reporting requirements.

The framework promotes:

* Consistent reporting processes
* Clear ownership and accountability
* Repeatable validation procedures
* Knowledge transfer across teams
* Sustainable reporting operations

This guide serves as a companion resource to the Generate product documentation and focuses on organizational processes, operational procedures, and reporting workflows.

***

## Relationship to Generate Documentation

[Generate user documentation](https://center-for-the-integration-of-id.gitbook.io/generate-documentation/user-guide/getting-started) explains how the application functions and how system components are configured for data migration in the [Generate developer guide](https://center-for-the-integration-of-id.gitbook.io/generate-documentation/developer-guides/getting-started).

This guide documents how your organization uses Generate to support reporting activities.

The guide focuses on:

* Source data preparation
* Report migration processes
* Validation activities
* Report generation procedures
* Submission preparation
* Historical reporting documentation
* Governance and accountability

***

## Scope

This documentation covers the complete reporting lifecycle, including:

1. Process and project setup
2. Technical setup and configuration
3. Reporting and validation activities
4. Historical reporting and issue tracking

Each Fact Type Reporting Guide follows the same structure to support consistency across reporting processes.
