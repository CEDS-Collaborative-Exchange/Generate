# Intended Audience

## Who Should Use This Guide

This documentation supports multiple roles involved in the reporting process.

| Audience               | Primary Responsibility                           |
| ---------------------- | ------------------------------------------------ |
| Program Staff          | Review and approve reporting results             |
| Data Owners            | Validate source system data                      |
| Technical Teams        | Maintain migration processes and reporting logic |
| Project Managers       | Coordinate reporting activities                  |
| Leadership             | Provide oversight and accountability             |
| Subject Matter Experts | Others that have system and data knowlage        |

***

## When to Use This Guide

Use this guide when:

* Implementing a new fact type
* Preparing for a reporting cycle
* Performing validation activities
* Investigating reporting issues
* Training new team members
* Reviewing historical submissions
* Documenting process changes

***

## How to Navigate This Documentation

The documentation is organized into five major areas.

### Reporting Overview

Provides shared guidance that applies across all reporting activities.

### Reporting Operations&#x20;

Provides detailed information for project resources

### Data Governance

Provides ownership, communication, and escalation information.

### Fact Type Reporting Guides

Provides detailed instructions specific to each fact type.

### System Data Diagrams (Comming Soon)

Provides architecture diagrams and reporting process flows.

***

## Recommended User Path

### New Users

1. Review the Reporting Overview section.
2. Review the Reporting Operations section.
3. Review the Fact Type Overview relevant to your work area.

### Returning Users

Navigate directly to the Fact Type Summary associated with the current reporting cycle.
