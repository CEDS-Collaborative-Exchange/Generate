# Communication Plan

## Purpose

Effective communication helps ensure reporting activities are coordinated across business and technical teams.

***

## Communication Events

| Event                   | Participants                    | Communication Method |
| ----------------------- | ------------------------------- | -------------------- |
| Reporting Kickoff       | All Stakeholders                | Meeting              |
| Source Data Ready       | Data Owners, Technical Team     | Email                |
| Migration Complete      | Technical Team, Validation Team | Email                |
| Validation Review       | Program Staff, Validation Team  | Meeting              |
| Final Approval          | Business Owners                 | Meeting              |
| Submission Confirmation | All Stakeholders                | Email                |

***

## Reporting Cycle Communications

During each reporting cycle, communication should include:

* Reporting schedules
* Data readiness status
* Migration status
* Validation results
* Issue resolution updates
* Submission status

***

## Meeting Cadence

| Meeting                    | Frequency           |
| -------------------------- | ------------------- |
| Reporting Planning Meeting | Annual              |
| Reporting Status Meeting   | As Needed           |
| Validation Review Meeting  | Per Reporting Cycle |
| Lessons Learned Review     | Post Submission     |
