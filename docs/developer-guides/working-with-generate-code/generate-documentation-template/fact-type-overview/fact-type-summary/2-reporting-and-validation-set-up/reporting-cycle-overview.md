# Reporting Cycle Overview

## Purpose

This section documents the operational procedures required to move data through the Generate reporting lifecycle.

The procedures described in this section should be followed during each reporting cycle to ensure reporting accuracy, consistency, and successful submission outcomes.

## Reporting Process

The reporting process follows a structured sequence of activities.

```
Source Data Preparation
        ↓
Source Validation
        ↓
Migration Processing
        ↓
Post Migration Validation
        ↓
Report Generation
        ↓
Report Validation
        ↓
Submission Preparation
        ↓
EDFacts Submission
```

## Validation Philosophy

Validation should occur throughout the reporting lifecycle rather than only at the end of the process.

Validation activities should confirm:

* Data completeness
* Data accuracy
* Business rule compliance
* Reporting rule compliance
* Submission readiness

## Reporting Deliverables

The reporting cycle produces:

* Validated warehouse data
* Report tables
* EDFacts submission files
* Reporting documentation
* Issue tracking updates
* Historical reporting records
